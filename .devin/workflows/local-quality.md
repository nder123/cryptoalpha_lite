# Цель
Стабильно получать одинаковое качество кода у всех разработчиков до CI.

# Шаги
1. Работай в отдельной ветке.
2. Установи зависимости backend:
   - `cd backend`
   - `poetry install --with dev`
3. Установи pre-commit хуки:
   - `poetry run pre-commit install`
4. Перед коммитом прогоняй:
   - `poetry run pre-commit run --all-files`
5. Локально прогоняй unit-тесты:
   - `poetry run pytest -q`

# Ожидаемый результат
- Форматирование/линтеры зелёные.
- Unit-тесты зелёные.
- Коммиты чистые и воспроизводимые.
