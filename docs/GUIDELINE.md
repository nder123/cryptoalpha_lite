# GUIDELINE для CryptoAlpha Lite

## Как работать с системой

### 1. Команды для Cascade в Windsurf
| Команда | Что делает |
|---------|------------|
| `СПЛАНИРУЙ: задача` | Создаёт план в scratchpad.md |
| `КОДИРУЙ: задача` | Выполняет следующий шаг из плана |
| `ТЕСТЫ: задача` | Запускает тесты и записывает результат |

### 2. Файлы проекта
- `.cascaderules` - правила для Cascade (в корне)
- `docs/scratchpad.md` - бортовой журнал (все агенты пишут сюда)
- `docs/GUIDELINE.md` - этот файл (документация)
- `windsurf_codex_bridge.sh` - скрипт для Codex CLI

### 3. Процесс работы
1. Пишем `СПЛАНИРУЙ: ...` в чат Cascade
2. Смотрим план в scratchpad.md
3. Если есть параллельные задачи - запускаем bridge-скрипт
4. Пишем `КОДИРУЙ: ...` для выполнения следующих шагов
5. Пишем `ТЕСТЫ: ...` для проверки

После любого изменения в коде/конфиге/юнитах systemd нужно сразу фиксировать контекст, чтобы не терять историю.

Куда писать:
- `docs/scratchpad.md` (раздел "Прогресс выполнения")

Что записывать (минимум):
